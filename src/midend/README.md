# 中端 (Midend) —— 负责人 C

符号表 + 语义分析 + 类型检查 + 作用域。语言规格见
[`docs/toyc-spec.md`](../../docs/toyc-spec.md)；交付给后端的 AST + 符号信息契约
（变量解析到「全局标签/局部槽位」、常量编译期求值等）见
[`docs/backend-design.md`](../../docs/backend-design.md) §1。

- **符号表**：记录变量/函数的名字、类型、作用域、存储位置。
- **语义分析**：检查未声明、重定义、类型不匹配等语义错误。
- **类型检查**：表达式/赋值/函数调用的类型一致性。
- **作用域**：嵌套作用域的进入/退出与名字解析。
- 输出：带符号信息的 AST / IR，交给后端。
- 头文件接口放在 `include/midend/`。

> 已实现：符号表 + 语义分析 + 类型检查 + 作用域检查，`check()` 返回 `CheckedAST`。
> 联调时补充了全局变量静态初值求值、放宽了 `while(常量真)` 的返回覆盖判定，详见
> `docs/integration-notes.md`。
