# 后端设计说明 (D)

> 后端负责：**RISC-V32 代码生成 + 优化 + 联调**。
> 目标：把中端(C) 交付的、经语义检查的 AST（带符号信息）翻译成正确的 **RV32 汇编**，
> 从 stdin 读源程序、向 stdout 写汇编，使 `main` 返回值（退出码）与等价 C 程序一致。
> 语言规格见 [`toyc-spec.md`](./toyc-spec.md)。

---

## 0. 关键决策速览

| 项 | 决策 |
| --- | --- |
| 目标 ISA | **RV32IM**（乘除模用 M 扩展的 `mul/div/rem`），ABI = **ilp32** |
| 数据类型 | 只有 32 位有符号 `int`（4 字节） |
| 翻译方式 | **AST 直接翻译**（tree-walking codegen）；未引入线性 IR，优化分为 AST 优化器和代码生成阶段优化 |
| 求值模型 | 表达式结果落在 **`a0`**；局部/形参优先占用 `s1–s11`，二元运算临时值优先用 `t0–t5`，不足时溢出到栈 |
| 调用约定 | 标准 RISC-V ABI：`a0–a7` 传参，`a0` 返回值，`ra` 返回地址，`sp` 16 字节对齐 |
| I/O | 无（ToyC 无 I/O）；程序结果 = `main` 返回值 |

> 实现现状：功能正确性已达成（功能用例全部通过），并在此基础上完成寄存器分配与多项优化
> 以提升性能（任务书今年提高了性能占比）。实现细节见 [`integration-notes.md`](./integration-notes.md) §4。

## 1. 输入接口（前端/中端 → 后端 的契约）

后端不直接接触源码，只消费中端交付的 **AST + 符号信息**。为使三方能并行开发，约定如下
最小契约（B 定义 AST 结点，C 完成语义检查，D 只读消费并自行做名字解析与存储分配）。
各项接口的最终决策见 §6。

### 1.1 顶层
- `Program`：全局声明（`GlobalVar` / `GlobalConst`）列表 + 函数（`FuncDef`）列表。
- `FuncDef`：`name`、`retType`(`int`/`void`)、`params`（名字+槽位）、函数体 `Block`、
  以及 C 计算好的 **`frameSize`（局部变量区总字节数）** 或供 D 自行统计的局部变量清单。

### 1.2 符号信息（中端 C 负责填充，后端只读）
每个变量/常量引用结点需能解析到唯一的存储位置，二选一：

- **全局**：`{ kind: Global, label: "g_<name>" }` —— 后端在 `.data`/`.sdata` 放一个 4 字节 word。
- **局部/形参**：`{ kind: Local, slot: <int> }` —— 后端据此分配 `s0(fp)` 相对偏移。

对**常量**（`const`），中端应完成编译期求值并给出 `constValue`，后端可直接用立即数，
无需为其分配存储（除非它被取地址——ToyC 没有取地址，故常量可完全内联）。

> 关键点：**后端要求「作用域已解析」**——同名遮蔽、声明先后、块作用域的问题都由 C 处理完，
> 交给 D 的每个 `VarRef` 都已明确指向某个 `slot` 或 `label`。D 不再做名字解析。

### 1.3 表达式/语句结点（后端需要能区分的种类）
- 表达式：`Num(const)`、`VarRef(符号)`、`Call(name,args)`、
  `Unary(op ∈ {+,-,!}, e)`、`Binary(op, l, r)`（算术 `+ - * / %`、关系 `< > <= >= == !=`、
  逻辑 `&& ||`）。
- 语句：`Block(stmts)`、`Empty`、`ExprStmt`、`Assign(target,expr)`、
  `LocalDecl`、`If(cond,then,else?)`、`While(cond,body)`、`Break`、`Continue`、
  `Return(expr?)`。

> 最终未引入线性 IR，采用 AST 直译以保持接口简单（见 §6）。

## 2. 代码生成 (codegen)

### 2.1 栈帧布局（每次进入函数）
被调用者保存 `ra`/`s0`，用 `s0` 作帧指针。栈向低地址增长，`sp` 保持 16 字节对齐。

```
高地址
        ┌───────────────────────────┐
        │ 调用者传入的第 9+ 个参数     │  ← 溢出参数（>8 个时；ToyC 一般用不到）
  s0 →  ├───────────────────────────┤
        │ 保存的 ra                  │  [s0 - 4]
        │ 保存的 s0(旧 fp)           │  [s0 - 8]
        ├───────────────────────────┤
        │ 局部变量 / 临时溢出区        │  [s0 - 12] ... 向下
        ├───────────────────────────┤
  sp →  │ (为被调函数预留的传参区)     │
        └───────────────────────────┘
低地址
```

序言 / 尾声（示意）：
```asm
func:
    addi  sp, sp, -FRAME       # FRAME 向上取整到 16 的倍数
    addi  t0, sp, FRAME
    sw    ra, -4(t0)
    sw    s0, -8(t0)
    mv    s0, t0               # s0 指向帧顶
    ...                        # 函数体
.func_epilogue:
    lw    ra, -4(s0)
    mv    sp, s0
    lw    s0, -8(s0)
    ret
```
- 入参 `a0..a7` 在序言中就位到各自的存储（`s` 寄存器或栈槽）；第 9 个及以后从调用者栈取。
- `return e;`：先算 `e` 到 `a0`，再 `j .func_epilogue`。`void` 函数尾部补一个到 epilogue 的跳转。
- 当帧大小超过 12 位立即数范围时，序言和尾声回退到 `li`、`sub`、`add` 的通用形式。

### 2.2 表达式求值
- 约定：每个表达式求值后结果在 **`a0`**。
- 二元运算 `l op r`：算 `l`→`a0`，把左值放入临时寄存器 `t0..t5`（当右子树不含函数调用且嵌套
  深度 < 6）或溢出到栈临时槽（否则，保证跨调用安全）→算 `r`→`a0`→按 `op` 生成指令（结果回 `a0`）。
- `-opt` 下，两个操作数均为寄存器变量时直接以这两个寄存器发射运算；比较条件中的寄存器与常量也
  直接分支，避免 `a0`/`t0` 的中转。
- `x = x +/- C` 且 `C` 可编码为 12 位立即数时直接生成 `addi x, x, C`。
- 算术：`add/sub/mul/div/rem`（RV32M）。除法/取模的符号语义与 C 的截断除法一致（`div`/`rem` 即为有符号）。
- 关系：用 `slt/sltu` + `xori/seqz/snez` 组合合成 6 种比较，结果规约为 0/1。
- 一元：`-` → `sub zero`/`neg`；`!` → `seqz`；`+` → 空操作。

### 2.3 逻辑与短路（`&&` / `||`）
不当普通二元运算，**用控制流实现短路**并把结果规约为 0/1：
```
a && b:  eval a; beqz a, Lfalse; eval b; snez a0,b; j Lend; Lfalse: li a0,0; Lend:
a || b:  eval a; bnez a, Ltrue ; eval b; snez a0,b; j Lend; Ltrue : li a0,1; Lend:
```
在 `if`/`while` 的条件位置，进一步做**跳转短路 + 落空布局**（`genCondJump(cond, target, jumpWhenTrue)`），
生成到目标标签的单条条件分支，另一侧顺序落下，省去物化 0/1 与多余的无条件跳转；关系运算按需取反以复用单条分支。

### 2.4 控制流
- `if (c) T`：`genCondJump(c 为假 → Lend); T; Lend:`（条件为真顺序落入 `T`）。
- `if (c) T else E`：`genCondJump(c 为假 → Lelse); T; j Lend; Lelse: E; Lend:`。
- `while (c) B`（**循环旋转**）：`j Lcond; Lbody: B; Lcond: genCondJump(c 为真 → Lbody); Lend:`。
  条件下沉到循环底部，稳态每轮迭代仅一条“为真回跳”分支；维护 `(Lcond, Lend)` 循环标签栈供
  `continue → j Lcond`、`break → j Lend` 使用。
- 标签用函数内自增计数器生成唯一名（如 `.L<func>_<n>`）。

### 2.5 函数调用
- 实参从左到右求值，前 8 个放 `a0..a7`，多余的压栈（ToyC 通常 ≤8）。
- `-opt` 对单参数调用直接保留 `a0` 中的求值结果，不创建临时栈槽；多参数调用仍使用保守的暂存方案，
  从而保持从左到右求值和跨调用安全。
- `call <name>`（= `jal ra, name`）后返回值在 `a0`。
- 跨调用存活的值都在被调用者保存寄存器（`s1–s11`）或栈上，`t` 系临时值不跨调用，故调用天然安全。

### 2.6 全局与常量
- 全局变量：`.data` 段 `g_<name>: .word <init>`（初始化表达式为编译期常量，中端已求值）。
  访问用 `la t0, g_<name>; lw/sw`。
- 常量：内联为立即数，不占存储。

## 3. 寄存器分配 (regalloc)

已实现的分配策略（内建在 `codegen`，非独立 pass）：
1. **局部变量/形参 → `s1–s11`**（被调用者保存，跨调用自动保持），超过 11 个的回落到栈槽；
   用到的 `s` 寄存器在序言/尾声成对保存恢复。
2. **表达式临时值 → `t0–t5`**：二元运算右子树不含函数调用且嵌套深度 < 6 时左操作数留在
   `t` 寄存器，否则溢出到栈临时槽以保证跨调用安全。
3. **叶子函数**（函数体不含任何调用）省去 `ra` 的保存与恢复。**叶子无栈帧**：若叶子函数局部数 ≤ 7
   且表达式嵌套浅，则把全部局部改放调用者保存寄存器 `a1..a7`，完全省去栈帧与序言/尾声——由于只用
   调用者保存寄存器、从不触碰 `s0..s11`，天然保持被调用者保存寄存器不变，是热点小函数最大的收益来源。

> 可选的进一步提升：活跃变量分析 + 线性扫描/图着色以跨基本块复用与合并寄存器。

## 4. 优化 (opt)

按「实现成本 / 收益」排序增量启用（`-opt` 打开）：
1. **常量折叠 + 常量传播**：编译期算掉常量子表达式（中端已折 `const`，这里覆盖更广）。
2. **寄存器直连**：两端均为寄存器变量时，直接发射 RV32 算术/关系指令；寄存器-常量比较直接装入
   一个临时寄存器并分支。
3. **自更新归约**：把 `x = x +/- C` 降为 `addi`，避免通用表达式求值的多次搬运。
4. **单参数直传**：单参数调用省去参数临时槽的存取；多参数和嵌套调用保持原有安全路径。
5. **小栈帧序言/尾声**：在 12 位偏移范围内使用 `addi` 形式，否则回退到通用大帧实现。
6. **强度削弱与窥孔清理**：乘 2 的幂改为移位，删除 `mv x,x`、相邻标签跳转和紧邻的冗余加载。
7. **跳转短路 + 落空布局 + 循环旋转**：条件生成单目标分支、常见路径顺序落下，`while` 条件下沉到底部（见 2.4）。
8. **叶子函数无栈帧**：叶子函数局部改放 `a1..a7`，消除每次调用的序言/尾声访存（见 §3）。
9. **小叶子函数内联**：不含调用（排除递归）、局部+形参 ≤ 8、函数体规模小的函数在调用点展开。
   实参直接写入形参寄存器，函数体在调用者栈帧上就地生成（临时槽平移避开在用槽位），
   `return` 改写为跳到展开结束标签；生成期间作用域栈只保留全局层+形参层，防止调用者局部被误解析。
   热点小函数的 `call/ret` 与参数搬运全部消失，并让内联体享受调用者的常量提升。
10. **循环条件常量提升**：把 `while` 条件（及循环内 `if` 条件）中频繁比较的非零字面量装入
    空闲的 `s` 寄存器，每个函数只装载一次；循环每轮省一条 `li`。`s` 寄存器跨调用保持，
    对含调用的循环体同样有效。
11. **安全表达式化简和控制流裁剪**：生成阶段识别单位元、零元、相同纯表达式的减法/比较，
    并删除纯表达式语句、常量 `if` 的不可达分支和 `while (0)`。涉及函数调用时仍完整求值，
    因而不改变副作用或短路语义。
12. **自尾递归消除**：仅将 `return f(args)` 中的当前函数 `f` 转成“实参暂存 → 回填形参 →
    跳回函数入口”；实参先暂存保证 `f(b, a)` 一类调用不覆盖旧值。非尾递归仍使用普通调用。
13. **局部前向常量传播**：局部变量的常量初值和直线赋值参与后续求值；未知分支、循环和新作用域
    会清除可变值事实，保证不跨控制流错误传播。
14. **相邻纯公共子表达式复用与块存储复用**：仅复用相邻声明/赋值中同构、无调用的右值；控制流、
    调用、作用域切换或不匹配赋值均使缓存失效。块作用域退出后重用其局部位置，降低寄存器溢出。
15. **AST 级优化器**（`src/backend/opt/Optimizer.cpp`，语义检查后、代码生成前运行）：
    ToyC 无指针/数组、别名分析平凡，因此以下经典中端优化可以安全实现，各 pass 迭代至不动点：
    - **全局常量传播**：全程未被赋值的全局以常量初值内联，初始化链迭代折叠；
    - **局部常量/复写传播**：带作用域的顺序数据流，`if` 分支交集合并、循环按赋值集合保守失效、
      同名遮蔽按作用域处理；
    - **常量折叠 + 代数化简**：32 位回绕求值；`x+0`、`x*1`、`x*0`（纯）、`x-x`（纯）等恒等式；
      `(x+c1)+c2` 重结合；`0&&e`/`1||e` 按短路语义定值；
    - **公共子表达式消除**：线性区间内纯右值复用此前局部持有的相同表达式；
    - **死代码删除**（mark-sweep）：以 `return`/全局赋值/调用为根的活跃标记，删除结果从未被读
      的纯语句与**整个无效循环**（`while(1)` 常量真条件不删）；
    - **循环不变量外提**：纯不变子表达式提升为循环前的 `@licmN` 临时量，跨出现点去重；
      循环体含调用时全局不视为不变量；
    - **常量分支消除**：`if(常量)` 以选中分支替换、`while(0)` 删除。
16. **循环内算术常量提升**：乘/除/模的任意非零常量与加/减的超立即数常量也提升进空闲寄存器
    （非叶子用 `s*`、叶子无栈帧函数用剩余 `a*`）。
17. **操作数直连**：二元运算的左/右操作数为寄存器变量、`zero` 或已提升常量时直接作为指令
    操作数，`x = y op z` 单指令直达目标寄存器；`+`/`-`/`<` 的 12 位立即数用 `addi`/`slti`。
18. **尾递归零访存回填**：无调用且表达式浅的自尾递归，新实参经 `t0..t2` 集中求值后统一回填
    形参（恒等实参跳过），不再经栈临时槽。

> 优化必须**保持语义**（尤其短路求值、除零行为、`return` 覆盖）。每加一个 pass 都要跑回归。

## 5. 联调与测试

- **对接顺序**：先与 C 敲定 §1 的 AST/符号契约 → 打通最小用例 → 逐类特性扩展。
- **里程碑（后端内部）**：
  1. `int main(){ return N; }` → 能在 RV32 上返回正确退出码。
  2. 算术/关系/一元 + 局部变量 + 栈帧。
  3. `if`/`while`/短路/逻辑。
  4. 函数定义与调用（含递归）。
  5. 全局变量/常量。
  6. `opt/` 优化 pass。
  7. 整机联调 + 回归。
- **验证方法**：`riscv64-linux-gnu-gcc -march=rv32im -mabi=ilp32 -static out.s -o a.out`
  → `qemu-riscv32 ./a.out; echo $?`，与等价 C 程序的退出码逐一比对（脚本放 `tests/scripts/`）。

最近一次 WSL2 验证中，42 个功能用例在普通和 `-opt` 模式下均通过。性能用例 `prime.tc` 的
退出码与 GCC `-O2` 一致（均为 192）；`perf_compare.sh` 默认执行 3 次，取最小得到本项目
`0.21s`、GCC `-O2` `0.18s`，性能比为 `0.86`。功能回归覆盖语言特性和已有优化路径；专项
性能模式由在线评测进一步衡量，未随仓库提交的临时对抗或压力基准不作为可复现结论。

## 6. 接口决策（已确认）

联调后各项均已定稿：

- **AST 结点与所有权**：前端 B 用 `std::unique_ptr` 拥有整棵 AST；中端 `SemanticAnalyzer::check`
  只做检查、返回 `CheckedAST{root}`（只读裸指针）；后端只读消费，不接管所有权。
- **局部变量槽位**：中端不下发 `slot` 号；由后端 D 遍历函数体自行统计并分配（寄存器或栈偏移）。
- **常量**：一律在编译期求值并内联为立即数、不占存储（后端 `tryEvalConst` 覆盖字面量、已声明
  常量及其算术/逻辑组合）；全局普通变量的静态初值同样求值后写入 `.data`。
- **线性 IR**：不引入，保持 AST 直译；优化分为语义检查后的 AST 优化器和代码生成阶段优化。
- **负数字面量**：词法只识别无符号整数，`-` 作为一元运算符；`-常量` 交由常量折叠处理
  （字面量以 `stoll` 截断到 32 位以容纳 `-2147483648` 边界值）。
