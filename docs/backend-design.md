# 后端设计说明 (D)

> 本文档记录后端（RISC-V 代码生成 + 优化 + 联调）的设计决策。
> 当前为占位提纲，随实现推进逐步补全。

## 1. 输入接口

- 来源：中端 (C) 交付的 AST / IR + 符号表。
- 待确认字段：
  - 变量：类型、作用域层级、栈偏移 / 全局标签
  - 函数：参数列表、返回类型、局部变量总大小
  - IR 指令集（若采用线性 IR）

## 2. 代码生成 (codegen)

- 遍历策略：AST 直接翻译 / 先降到 IR 再选指令（待定）
- 栈帧布局：`ra`、`s0(fp)`、局部变量区、参数溢出区
- 调用约定：RISC-V 标准 ABI（`a0-a7` 传参，`a0` 返回值，`sp` 16 字节对齐）

## 3. 寄存器分配 (regalloc)

- 第一版：全部溢出到栈 / 简单线性扫描
- 进阶：活跃变量分析 + 图着色（可选）

## 4. 优化 (opt)

- 常量折叠 / 常量传播
- 死代码消除
- 代数化简
- 窥孔优化

## 5. 联调与测试

- 与前端/中端逐层对接，先跑通最小用例再扩展
- 验证：`riscv64-linux-gnu-gcc` + `qemu-riscv64`，比对退出码 / stdout
- 回归测试脚本放在 `tests/scripts/`

- [ ] 确定 RISC-V 位宽（RV32 / RV64）
