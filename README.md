# mini-compiler-riscv

程序语言理论与编译技术 —— 编译系统实践（课程大作业）

一个用 **C++** 实现的小型编译器：将 C 语言子集（SysY，具体文法以任务书为准）源程序
编译为 **RISC-V** 汇编代码。整体流程遵循经典编译器结构：

```
源程序 (.sy/.c)
   │  ┌──────────── 前端 (B) ────────────┐
   ▼  │ 词法分析 → 语法分析 → AST 构建     │
        └───────────────┬──────────────────┘
                        ▼
        ┌──────────── 中端 (C) ────────────┐
        │ 符号表 → 语义分析 → 类型检查 → 作用域 │
        └───────────────┬──────────────────┘
                        ▼
        ┌──────────── 后端 (D) ────────────┐
        │ RISC-V 代码生成 → 优化 → 联调       │
        └───────────────┬──────────────────┘
                        ▼
                目标汇编 (.s)  →  riscv 工具链汇编/链接/运行
```

## 团队分工

| 成员 | 模块 | 职责 |
| --- | --- | --- |
| A | 报告 / 测试 | 实验报告、整理截图、设计测试用例、协助调试 |
| B | 前端 | 词法分析 + 语法分析（AST 构建） |
| C | 中端 | 符号表 + 语义分析 + 类型检查 + 作用域 |
| D | 后端 | RISC-V 代码生成 + 优化 + 联调 |

> 本仓库骨架由 D（后端）搭建。各模块的具体接口以联调约定为准，见 `docs/`。

## 构建（规划）

项目使用 **C++** 编译，计划采用 CMake 组织构建：

```bash
cmake -B build -S .
cmake --build build -j
# 生成的编译器可执行文件：build/compiler
./build/compiler input.sy -o output.s
```

> 目前处于骨架阶段，各模块尚未实现，`CMakeLists.txt` 待接口稳定后补全。

## 目标平台工具链

后端生成 RISC-V（RV32/RV64，以任务书为准）汇编，验证时使用：

```bash
riscv64-linux-gnu-gcc / riscv64-unknown-elf-gcc   # 汇编 + 链接
qemu-riscv64 / spike + pk                          # 运行
```
