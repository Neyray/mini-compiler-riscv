# 后端 (Backend) —— 负责人 D

后端负责把中端产出的、经过语义检查的中间表示（AST / IR）翻译成 **RISC-V 汇编**，
并在此基础上做优化与整机联调。

## 子模块

| 目录 | 模块 | 说明 |
| --- | --- | --- |
| `codegen/` | 代码生成 | 遍历 AST / IR，生成 RISC-V 指令序列；处理函数调用约定、栈帧、全局变量、数组寻址 |
| `regalloc/` | 寄存器分配 | 从虚拟寄存器到物理寄存器的分配（先做简单线性扫描 / 栈分配，后续可上图着色） |
| `opt/` | 优化 | 常量折叠、死代码消除、代数化简、窥孔优化等（按实现顺序增量添加） |

## 与其它模块的接口约定（联调）

后端的输入来自中端 (C)。接口在联调前需与 B/C 共同确认，初步约定：

- **输入**：中端交付的 AST（或线性 IR）+ 符号表信息（变量/函数的类型、作用域、偏移）。
- **输出**：一个 `.s` 汇编文件，可被 `riscv64-*-gcc` 直接汇编链接。
- 头文件接口放在 `include/backend/`，实现放在本目录。

> 具体的 IR 数据结构、符号表字段一旦确定，更新到 `docs/backend-design.md`。

## 计划实现顺序

1. 打通「最小可运行」：`int main(){ return N; }` → RISC-V，能在 qemu 上返回正确退出码。
2. 表达式与算术运算、局部变量与栈帧。
3. 控制流：`if / while`、短路求值、逻辑运算。
4. 函数定义与调用（RISC-V 调用约定：a0–a7 传参、ra/sp 维护）。
5. 数组与全局变量。
6. 优化 pass（`opt/`）。
7. 与前端/中端整机联调 + 回归测试。

## 目标平台约定

- 指令集：RISC-V（RV64GC / RV32，以任务书为准）
- 调用约定：标准 RISC-V ABI（`sp` 16 字节对齐，`a0` 返回值，`ra` 返回地址）
- 验证：`riscv64-linux-gnu-gcc` 汇编链接 + `qemu-riscv64` 运行，比对退出码/输出

## 状态

骨架阶段，等待中端 (C) 接口确定后开始编码。当前目录仅含占位文件。
