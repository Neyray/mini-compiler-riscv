# mini-compiler-riscv

程序语言理论与编译技术 —— 编译系统实践（课程大作业）

一个用 **C++** 实现的小型编译器：将 **ToyC 语言**（C 语言的一个简化子集，具体文法见任务书
与 `docs/toyc-spec.md`）源程序编译为 **RISC-V32 汇编代码**。整体流程遵循经典编译器结构：

```
ToyC 源程序 (.tc)  ← 从标准输入 stdin 读取
   │  ┌──────────── 前端 (B) ────────────┐
   ▼  │ 词法分析 → 语法分析 → AST 构建     │
        └───────────────┬──────────────────┘
                        ▼
        ┌──────────── 中端 (C) ────────────┐
        │ 符号表 → 语义分析 → 类型检查 → 作用域 │
        └───────────────┬──────────────────┘
                        ▼
        ┌──────────── 后端 (D) ────────────┐
        │ RISC-V32 代码生成 → 优化 → 联调     │
        └───────────────┬──────────────────┘
                        ▼
          RISC-V32 汇编 (.s)  → 写入标准输出 stdout
```

> **接口约定（任务书要求）**：编译器从 **stdin** 读取 ToyC 源程序，向 **stdout** 写入
> RISC-V32 汇编。stderr 输出随意。性能测试时会传入 `-opt` 参数表示开启优化（可选择启用优化或忽略）。

## 构建

```bash
# 方式一：CMake（生成 build/compiler）
cmake -B build -S .

make
```

## 运行接口（任务书约定）

编译器从 **标准输入** 读取源程序，向 **标准输出** 写出汇编。本地测试用重定向即可：

```bash
./build/compiler < input.tc > output.s          # 功能编译
./build/compiler -opt < input.tc > output.s      # 开启优化（性能测试）
```

## 目标平台工具链

后端生成 **RISC-V32（RV32）** 汇编。本地验证（RV32 ILP32）时可用：

```bash
# 汇编 + 链接为 RV32 可执行文件
riscv64-linux-gnu-gcc -march=rv32im -mabi=ilp32 -static output.s -o a.out
# 运行并检查退出码（程序结果 = main 的返回值，0~255）
qemu-riscv32 ./a.out ; echo "exit code = $?"
```

> 也可使用 `riscv32-unknown-elf-gcc` + `spike`/`pk`。评测系统以 `gcc -O2` 生成代码的运行时间为性能基准。

## 本地回归测试

ToyC 是 C 的子集，可用本机 `gcc` 直接把 `.tc` 当作 C 编译运行，得到每个用例的期望退出码作为

> freestanding 方式链接。评测环境使用其自带 rv32 工具链，本编译器输出为标准汇编，无需该文件。

## `-opt` AST 级优化器（中端优化 pass）

`-opt` 下，语义检查之后、代码生成之前会先运行一个 AST 级优化器
（`src/backend/opt/Optimizer.cpp`）。ToyC 无指针/数组、别名分析平凡，因此以下经典优化
可以安全实现，各 pass 迭代至不动点：

- **全局常量传播**：全程未被赋值的全局变量以其常量初值内联到所有读取点，全局初始化链
  （`int g2 = g1 + 3;`）迭代折叠。
- **局部常量/复写传播**：带作用域的顺序数据流。`int b = a; int c = b;` 的读取直接替换为
  源变量或常量；`if` 两分支按交集合并事实，循环内被赋值的名字保守失效，同名遮蔽按作用域处理。
- **常量折叠与代数化简**：32 位回绕语义求值常量子表达式；`x+0`、`x*1`、`x*0`（纯）、`x/1`、
  `x%1`（纯）、`x-x`（纯）、双重取负等恒等式；`(x+c1)+c2` 重结合；常量规范化到右侧；
  `0&&e`/`1||e` 按短路语义定值（`e` 本就不执行，可安全丢弃）。
- **公共子表达式消除**：线性区间内，声明/赋值的纯右值与此前某局部持有的表达式相同时直接复用
  该变量；构成变量被改写、遇分支/循环/调用边界即失效。
- **死代码删除**（mark-sweep）：以 `return`、全局赋值、函数调用为根做活跃标记，不动点迭代后
  删除结果从未被读取的纯语句——包括**整个无效循环**（C 前进性保证允许；常量真条件的
  `while(1)` 不删）。
- **循环不变量外提**：把循环内所有变量均不在循环中被改写的纯子表达式（含乘/除/模或多个运算）
  提升为循环前的编译器临时量 `@licmN`，同一表达式跨出现点去重（兼具跨语句 CSE 效果）；
  循环体含调用时全局变量不视为不变量。
- **常量 `if`/`while(0)` 消除**：条件为常量的分支直接以选中分支替换，`while(0)` 整体删除。

## `-opt` 代码生成优化

在 AST 优化器之后，代码生成层继续为常见热路径启用以下语义等价的优化：

- **小叶子函数内联**：不含任何调用（天然排除递归）、局部与形参不超过 8 个且函数体规模小的函数，在调用点直接展开——实参写入形参的寄存器，函数体在调用者栈帧上就地生成，`return` 改写为跳转到展开结束标签。热点小函数的 `call/ret` 与参数搬运开销完全消失。
- **循环条件常量提升**：把循环中频繁比较的字面量（如 `while (i < 4000)` 的边界）装入空闲的被调用者保存寄存器 `s*`，每个函数只装载一次；循环每轮省去一条 `li`。`s` 寄存器跨调用保持，对含调用的循环体同样有效。
- **叶子函数无栈帧**：函数体不含任何调用、局部数 ≤ 7 且表达式嵌套浅时，把全部局部分配到调用者保存寄存器 `a1..a7`，**完全省去栈帧与序言/尾声**。此类函数不触碰任何被调用者保存寄存器，天然保持其不变，因而每次调用都不再产生保存/恢复的访存开销。
- **条件落空布局 + 循环旋转**：`if`/`while` 的条件生成到目标标签的单条分支，常见路径顺序落下，省去多余的无条件跳转；`while` 将条件测试下沉到循环底部，稳态每轮迭代只剩一条“为真回跳”分支。
- **常量/寄存器直写**：声明与赋值的右值为常量或寄存器变量时，直接 `li`/`mv` 到目标寄存器，不再经 `a0` 中转。
- 两个操作数都已在寄存器中时（或一侧为 0、已提升常量时），直接发射算术、关系或比较指令，避免经由 `a0`/`t0` 的搬运。
- 将寄存器变量的 `x = x +/- 常量`（12 位有符号立即数范围内）直接降为 `addi`。
- 单参数调用直接把求值结果保留在 `a0` 传递，省去临时栈槽的存取。
- 小于等于 2047 字节的栈帧使用短序言与尾声；更大的栈帧自动回退到通用实现。
- **安全表达式化简与死语句删除**：化简 `x+0`、`x*1`、`x/-1`、`x%1` 等语义明确的模式；仅当两侧均不含调用时，才折叠 `x-x` 和相同表达式的比较。纯表达式语句、常量 `if` 分支和 `while (0)` 也直接删除，函数调用仍保留其求值次序与副作用。
- **自尾递归消除**：仅匹配函数末尾的 `return 当前函数(实参...)`。所有新实参先集中求值再统一回填形参并跳转到函数体入口，因此交换形参等情形也不会发生覆盖：无调用且表达式浅的常见情形用 `t0..t2` 寄存器中转（**零访存**），恒等实参（实参就是对应形参）直接跳过搬移；复杂情形退回栈临时槽的保守路径。非尾递归和间接递归继续走普通调用路径。
- **局部前向常量传播**：可变局部变量的常量初值和直线赋值会向后续表达式传播；未知分支和循环立即丢弃可变常量事实，避免把单一路径或单次迭代错误外推。
- **相邻纯公共子表达式复用**：相邻声明/赋值的同构、无函数调用右值复用前一次结果；遇到不匹配赋值、分支、循环、作用域切换或调用即失效，不进行跨基本块 CSE。
- **块作用域存储复用**：嵌套块退出后，后续互斥或顺序块复用其局部寄存器/栈槽，减少复杂函数中不必要的寄存器溢出。

- **二元运算操作数直连**：左/右操作数是寄存器变量、`zero` 或已提升常量时，直接以其寄存器
  作为指令操作数（含 `x = y op z` 单指令直达目标寄存器），消除经由 `a0`/`t*` 的搬运；
  加减 12 位立即数降为单条 `addi`/`slti`。
- **循环内算术常量提升**：除条件比较常量外，循环内乘/除/模的任意非零常量、加/减超出
  12 位立即数范围的常量也提升进空闲寄存器（非叶子函数用 `s*`，叶子无栈帧函数用剩余的
  `a*`），每轮迭代省去一条 `li`。

这些优化只在 `-opt` 下启用。AST 优化器 pass 之间的组合（如常量传播喂给 DCE、LICM 的
临时量再被复写传播清理）由多轮不动点迭代保证。

在 WSL2 + `qemu-riscv32` 上，`tests/scripts/perf_compare.sh` 对 `tests/cases/perf/prime.tc`
默认执行 3 次并取最小值。本次验证中，本项目 `-opt` 产物与 GCC `-O2` 产物退出码均为 `192`，
最佳用时分别为 `0.21s` 与 `0.18s`，性能比 `min(1, gcc-O2/ours)` 为 **`0.86`**。时间受
机器负载和 QEMU 版本影响，提交前应在目标环境复测。AST 优化器的专项模式覆盖由在线评测结果
与仓库中的功能回归共同验证；未随仓库提交的临时基准不作为可复现性能结论。

## 完成情况

- 前端：词法分析 + 语法分析 + AST（B）
- 中端：符号表 + 语义分析 + 类型检查 + 作用域（C）
- 后端：RISC-V32 代码生成（D）
- 中端 AST 优化器（`-opt`）：**全局常量传播**、**局部常量/复写传播**、**常量折叠与代数化简**、**公共子表达式消除**、**死代码删除（含无效循环）**、**循环不变量外提**、**常量分支消除**（D）
- 后端优化：寄存器分配、常量折叠/传播、短路跳转、强度削弱、窥孔、寄存器直连、`addi`/`slti` 立即数直达、单参数直传、小栈帧、**小叶子函数内联**、**循环常量提升（条件+算术，s/a 寄存器）**、**叶子函数无栈帧（a1..a7）**、**条件落空布局 + 循环旋转**、**操作数直连（目标寄存器直达）**、**自尾递归消除（t 寄存器零访存回填）**、常量/寄存器直写（D）
- 构建：CMake / Make 双入口，从 `stdin` 读源程序、向 `stdout` 写汇编，识别 `-opt`
- 联调 + 测试（D + A）：42 个功能用例在 `-opt` 与非 `-opt` 下全部通过；`prime.tc` 与 GCC `-O2` 结果一致；100,000 次自尾递归压力用例通过
- 实验报告（A）：见 `docs/编译系统实践报告.docx`
