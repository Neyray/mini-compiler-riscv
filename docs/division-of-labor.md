# 分工说明

| 成员 | 模块 | 具体职责 | 主要目录 |
| --- | --- | --- | --- |
| A | 报告 / 测试 | 实验报告、整理截图、设计测试用例、协助调试 | `tests/`, 报告文档 |
| B | 前端 | 词法分析、语法分析、AST 构建 | `src/frontend/`, `include/frontend/` |
| C | 中端 | 符号表、语义分析、类型检查、作用域 | `src/midend/`, `include/midend/` |
| D | 后端 | RISC-V 代码生成、优化、联调 | `src/backend/`, `include/backend/` |

## 协作约定

- 各成员在自己模块目录内工作，公共头文件放到对应的 `include/<module>/`。
- 模块间接口（前端→中端→后端的数据结构）在动手前统一约定，写入 `docs/`
  （语言规格见 `toyc-spec.md`，AST/符号→后端契约见 `backend-design.md` §1）。
- 使用 Git 分支协作：建议每人一个 feature 分支，联调时合并到 `main`。
- 构建统一用 CMake + C++（C++20，任务书允许的语言/版本之一）。

## 里程碑（均已完成）

1. 接口约定（AST 结构、符号表结构）—— 全员
2. 前端产出 AST —— B
3. 中端完成语义检查、交付经检查的 AST —— C
4. 后端生成正确的 RISC-V 汇编 —— D
5. 整机联调 + 优化 + 回归测试 —— D + A
6. 实验报告 —— A
