# ToyC 语言规格（任务书摘录，全员依据）

> 本文档摘录/整理自课程任务书《编译系统实践》，作为前端(B)、中端(C)、后端(D) 三方
> 共同的语言规格依据。如与任务书原文有出入，以任务书为准。

## 1. 概述

ToyC 是 C 语言的一个简化子集，为课程练习设计。相较于 C，ToyC **省去了数组、指针、
I/O、多文件编译**等特性，只有 `int`（32 位有符号整数）一种数据类型。ToyC 源文件可用
C 编译器直接编译，运行结果与等价 C 代码一致。

- **输入**：编译器从 **stdin** 读取 ToyC 源程序。
- **输出**：向 **stdout** 写出 **RISC-V32** 汇编。stderr 随意。
- **结果判定**：以 `main` 函数的返回值（进程退出码，0~255）为准。
- 所有测试用例均无语法错误、满足全部语义约束、不含未定义行为（编译器可不做健壮的错误恢复，
  但仍需正确处理合法程序）。
- 性能测试会传入 `-opt` 参数表示开启优化（可选）。

## 2. 终结符

| 终结符 | 说明 | 正则表达式 |
| --- | --- | --- |
| `ID` | 标识符（符合 C 规范） | `[_A-Za-z][_A-Za-z0-9]*` |
| `NUMBER` | 十进制整数常量 | `-?(0\|[1-9][0-9]*)` |

- 空白字符（空格、制表符、换行等）与注释一律忽略。
- 注释规范同 C：单行 `// ... 换行`；多行 `/* ... */`（到最近的 `*/` 结束）。

> 注意：`NUMBER` 的正则含可选前导 `-`。词法层面负号是否并入数字字面量、还是作为一元
> 运算符处理，需要 B（前端）与 C/D 统一约定，避免 `a-1` 之类的歧义（建议：把 `-` 交给
> 一元表达式 `UnaryExpr` 处理，`NUMBER` 只识别非负部分，见第 5 节说明）。

## 3. 文法（BNF，开始符号 CompUnit）

```
CompUnit   → (Decl | FuncDef)+
Decl       → ConstDecl | VarDecl
ConstDecl  → "const" "int" ID "=" Expr ";"
VarDecl    → "int" ID "=" Expr ";"
Stmt       → Block
           | ";"
           | Expr ";"
           | ID "=" Expr ";"
           | Decl
           | "if" "(" Expr ")" Stmt ("else" Stmt)?
           | "while" "(" Expr ")" Stmt
           | "break" ";"
           | "continue" ";"
           | "return" Expr? ";"
Block      → "{" Stmt* "}"
FuncDef    → ("int" | "void") ID "(" (Param ("," Param)*)? ")" Block
Param      → "int" ID
Expr       → LOrExpr
LOrExpr    → LAndExpr | LOrExpr "||" LAndExpr
LAndExpr   → RelExpr  | LAndExpr "&&" RelExpr
RelExpr    → AddExpr  | RelExpr ("<" | ">" | "<=" | ">=" | "==" | "!=") AddExpr
AddExpr    → MulExpr  | AddExpr ("+" | "-") MulExpr
MulExpr    → UnaryExpr| MulExpr ("*" | "/" | "%") UnaryExpr
UnaryExpr  → PrimaryExpr | ("+" | "-" | "!") UnaryExpr
PrimaryExpr→ ID | NUMBER | "(" Expr ")" | ID "(" (Expr ("," Expr)*)? ")"
```

> 运算符优先级/结合性由文法层次隐含：`|| < && < 关系 < 加减 < 乘除模 < 一元 < 基本`，
> 二元运算左结合，一元运算右结合，与 C 一致。

## 4. 语义约束

### 程序结构
- 程序由全局变量声明、全局常量声明、函数定义组成。
- 必须包含一个 `main` 函数：**参数列表为空、返回类型为 `int`**，作为入口点。

### 函数
- 参数与返回值只有 `int`（`void` 仅用于返回类型）。
- 函数只能声明在全局作用域，名称唯一，不能嵌套定义。
- 函数不能作为值存储、传递或运算。
- 调用必须写在被调函数声明**之后**（支持递归，即函数调用自身）。
- 返回 `int` 的函数：**每一条可能的执行路径都必须通过 `return` 返回一个 int**。
- 返回 `void` 的函数：可不含 `return`；若含，则 `return` 不能带返回值。

### 语句
- 语句包括：语句块、空语句、表达式语句、赋值、常量/变量声明、`if-else`、`while`、
  `break`、`continue`、`return`。语法语义与 C 一致。
- 没有返回值的函数调用表达式不能作为 `if`/`while` 条件或赋值语句的右值。
- `break`/`continue` 只能出现在循环中。

### 变量与常量
- 只支持 32 位有符号整数（`int`）局部/全局变量与局部/全局常量（`const`）。
- **所有声明必须带初始化表达式**（VarDecl 和 ConstDecl 都要）。
- **常量约束**：`const` 修饰的标识符为常量，其值在**编译期必须能确定**（初始化表达式只能
  由数字字面量、已声明常量及它们的算术/逻辑运算组成）；常量声明后不可修改（不能作为左值）。
- **作用域与生命周期**：全局变量/常量全程有效；局部变量/常量从声明点起、到所在作用域结束止；
  函数形参在函数体内可见；`Block` 创建嵌套作用域，内层屏蔽外层同名量；使用必须在声明之后。

### 表达式
- 支持逻辑、关系、算术、括号、变量/常量引用、字面量、函数调用。
- 运算符优先级、结合性、计算规则同 C。
- `&&`、`||` 遵循**短路求值**；非零为真、零为假；除数不能为零（合法程序不会出现除零）。

## 5. 对各模块的落地提示

- **前端(B)**：一元 `-`/`+`/`!` 由 `UnaryExpr` 处理；建议 `NUMBER` 词法只取非负数字串，
  负号交给 `UnaryExpr`，AST 中保留一元节点或在此处直接折叠为负字面量。
- **中端(C)**：负责常量求值（`const` 初始化必须编译期可算）、`return` 覆盖性检查、
  `break/continue` 位置检查、作用域解析并把每个变量绑定到「全局标签」或「函数内的槽位/偏移」。
- **后端(D)**：只有一种 4 字节类型，寄存器/栈分配简单；短路求值、比较、取模/除法、
  递归调用是主要难点，详见 `backend-design.md`。
